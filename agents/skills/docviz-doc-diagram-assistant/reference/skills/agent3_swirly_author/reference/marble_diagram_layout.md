# 弹珠图布局规范（硬约束 + 可读性）

本文件合并了：
- marble_diagram_layout.md（硬性约束 + 可读性增强）
- 用户反馈补充规则

**适用范围**：所有弹珠图（Swirly / JSON→SVG / 其他渲染器）。

---

## A. 硬性约束（必须满足）

### 约束 1：线与文字/框的绘制顺序（零遮挡）
- 先画底层形状（轨道、band、segment、event 圆点）与所有连线/箭头。
- 再统一画：band/segment/event 标签、算子框与框内文字、link 标签。
- 目标：线/箭头不得遮挡任何框或文字。

### 约束 2：Notes 与图主体不重叠
- Notes 区域位于图下方/上方独立区域，与主体留出间距。
- 必须按内容下边界 + gap + notes/legend 高度计算画布高度。

### 约束 3：Legend 不被线遮挡
- Legend 放独立区域并最后绘制，或在浅色底区域内绘制。

### 约束 4：轨道标签完整可见（左侧不裁切）
- 左边距足够，最长标签不可被裁切；必要时 viewBox 预留余量。

### 约束 5：标题与图不重叠
- 标题、时间轴、首条轨道分层布局，保持 25–30px 以上间距。

### 约束 6：图中用词引用文档原文
- title、lane、segment/event/band/算子、notes/legend 必须贴近原文。

### 约束 7：事件对齐与段边界对齐
- **事件**必须对齐同一时间轴。
- **段的起止边界**必须对齐相关事件；段不可为“对齐”而拉长。

### 约束 8：泳道间距自适应
- 泳道间距必须依据**上下泳道段高 + 弹珠半径 + 缓冲**自适应计算。
- 目标：段与段、段与弹珠不重叠。

### 约束 9：操作框位置
- 操作框必须位于两条泳道之间（`y_between`），不得压在泳道线上。
- 操作框绘制层级必须高于连线。

### 约束 10：可选事件符号使用范围
- 仅用于真正“可选事件”（如 B2 BuildMilestone）。
- 不用于 gate-filter；过滤由上下游事件缺失表达。

### 约束 11：渲染层级顺序
- 固定顺序：底线 → 段 → 连线 → 操作框 → 事件 → 标签。

---

## B. 可读性增强（强烈建议）

### 规则 1：侧栏统一宽度 + 自动换行
- 轨道标签区固定宽度。
- 标签过长时自动换行，不强行单行压缩或裁切。

### 规则 2：框与弹珠分层，框内标签在弹珠上方
- Segment/Band 作为背景层，弹珠作为前景事件点。
- 段内标签放在框内左上角，确保不与弹珠重叠。

### 规则 3：关系标签不夹在两条流中间
- Link 标签应避开上下泳道拥挤区，可上移/下移。
- 标签文本需带上下文（如 `Wall.StateChange`）。

### 规则 4：事件标签可读性
- 事件标签默认在上方；重叠则下移或移除。
- 长标签允许多行换行。

---

## C. 说明性约定

- 缩写映射必须在 Legend/Notes 给出（如 W1/W2/W3/W4）。
- Notes/Legend 必须始终可见（footer 位置需随自适应高度调整）。
